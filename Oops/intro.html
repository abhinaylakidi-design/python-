Object Oriented Programming System (OOPS) in Python

1. Introduction to OOPS

Object Oriented Programming System (OOPS) is a programming paradigm that organizes code using objects and classes. Python supports OOPS, making programs more structured, reusable, and easy to maintain.

Why OOPS?
  • Code reusability
  • Better structure
  • Easy maintenance
  • Real-world modeling

2. Class

A class is a blueprint for creating objects. It defines variables (attributes) and methods (functions). 

class Student:
    name = "Abhinay"
    age = 22

3. Object

An object is an instance of a class. It represents a real-world entity. 

s1 = Student()
print(s1.name)
print(s1.age)

4. Constructor (__init__)

A constructor is a special method used to initialize object data. It is automatically called when an object is created. 

class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age

s1 = Student("Abhinay", 22)
print(s1.name, s1.age)

5. self Keyword

The self keyword represents the current object. It is used to access instance variables and methods. 

class Demo:
    def show(self):
        print("Hello Python")

6. Encapsulation

Encapsulation is the process of wrapping data and methods together and restricting direct access to variables. 

class Bank:
    def __init__(self):
        self.__balance = 1000

    def get_balance(self):
        return self.__balance

7. Inheritance

Inheritance allows one class to acquire the properties of another class. It promotes code reuse. 

class Parent:
    def show(self):
        print("Parent class")

class Child(Parent):
    def display(self):
        print("Child class")

c = Child()
c.show()
c.display()

8. Polymorphism

Polymorphism means "many forms". The same method name can have different behaviors. 

class Dog:
    def sound(self):
        print("Bark")

class Cat:
    def sound(self):
        print("Meow")

for animal in (Dog(), Cat()):
    animal.sound()

9. Abstraction

Abstraction hides implementation details and shows only essential features. 

from abc import ABC, abstractmethod

class Vehicle(ABC):
    @abstractmethod
    def speed(self):
        pass

class Bike(Vehicle):
    def speed(self):
        print("Bike speed is 60 km/h")

10. Real-Time Example

Below is a simple real-world example using a Mobile class. 

class Mobile:
    def __init__(self, brand, price):
        self.brand = brand
        self.price = price

    def details(self):
        print(self.brand, self.price)

m1 = Mobile("Samsung", 20000)
m1.details()

11. Summary

  • Class is a blueprint
  • Object is an instance of a class
  • Encapsulation protects data
  • Inheritance reuses code
  • Polymorphism allows multiple behaviors
  • Abstraction hides complexity
