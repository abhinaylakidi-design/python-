Functions in Python – Complete Explanation

1. What is a Function?

A function in Python is a reusable block of code that performs a specific task. Instead of writing the same code again and again, we define it once and call it whenever needed.

✔ Functions help in reducing code duplication and improving readability.

2. Why Do We Use Functions?

  • Improves code reusability
  • Makes programs easy to understand
  • Reduces code length
  • Helps in debugging and maintenance

3. Syntax of a Function

The basic syntax of defining a function in Python is:

def function_name(parameters):
    statements
    return value
  • def → Keyword used to define a function
  • function_name → Name of the function
  • parameters → Inputs to the function (optional)
  • return → Sends output back to the caller (optional)

4. Function Without Parameters

This type of function does not take any input.

def greet():
    print("Hello, Welcome to Python Functions")

greet()
✔ The function greet() is called without passing any arguments.

5. Function With Parameters

Parameters allow us to pass data into a function.

def greet_user(name):
    print("Hello", name)

greet_user("Abhinay")

6. Function With Return Statement

The return statement sends a value back to the function caller. 

def add(a, b):
    return a + b

result = add(10, 20)
print(result)

7. Types of Functions in Python

a) Built-in Functions

Functions already provided by Python.

print("Hello")
len("Python")
type(10)

b) User-Defined Functions

Functions created by the programmer.

def square(num):
    return num * num

8. Default Arguments

Default arguments allow a function to use predefined values if no value is passed. 

def country(name, nation="India"):
    print(name, "is from", nation)

country("Abhinay")
country("Alex", "USA")

9. Keyword Arguments

Arguments passed using parameter names.

def student(name, age):
    print(name, age)

student(age=21, name="Abhinay")

10. Advantages of Functions

  • Modular programming
  • Better code organization
  • Easy testing and debugging
  • Improved readability
